15. CONFIDENCE BREAKDOWN SCORING LOGIC SUMMARY
--------------------------------------------------------------------------------
- HTF Alignment: base 50; +/-20 internal trend vs setup; +/-20 trend vs setup;
  +10 if RSI agrees with direction.
- Zone Quality: OB quality score/label -> else Fib leg score -> else signal
  strength (reason states when derived from strength).
- Structure Confluence: 35 + 22*(extra confluent signals) + bonuses
  (FVG confluence / high-prob filter / absorption / untouched FVG).
- Momentum: 50 + (RSI deviation)*1.5 + (5-bar ROC)*4, direction-aware
  (NOT raw RSI).
- Volume: 50 + (relVol-1)*40 from real 20-bar relative volume; "unavailable"
  when volume data is missing (never a flat 50).
- Status from score: pass>=75, medium>=55, weak>=35, fail<35, else unavailable.

20. KNOWN RISKS
--------------------------------------------------------------------------------
- Embedded TradingView widget needs outbound access to s.tradingview.com; if the
  environment network policy blocks it the iframe is blank (Open-in-TradingView
  still works).
- The Dashboard view auto-polls /api/scan etc. while open (pre-existing) and thus
  consumes tokens; left as-is to avoid changing Dashboard design, but flagged as
  it intersects the "no token spend without intent" rule.
- Severity inference for free text is keyword-based; unusual backend status
  strings fall back to neutral "info" (never a false green).
- Top-nav -> tab mapping is a best-effort assumption; adjust if your tab
  semantics differ (e.g., Watchlist should map to watchlist rather than pairs).
