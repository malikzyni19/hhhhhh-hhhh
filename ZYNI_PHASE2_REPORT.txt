3. JS FUNCTIONS REPLACED
================================================================================

REMOVED:
  renderLiveMonitorInto  — old version (simple grid with lmAdd input + lmGrid div)
  _lmAdd                 — added symbol to ST.stars via text input (no longer needed)
  _lmRemove              — removed from ST.stars (legacy path, superseded by DB delete)
  _lmPaint               — old card painter (rendered _lmCache pairs, not DB items)
  (All 4 were only called from within the old LM system — no other callers)

ADDED:
  renderLiveMonitorInto(scroll)
    - Adds .lm-active to scroll element (full-height layout)
    - Sets scroll.innerHTML to locked UI shell:
        .lm-outer > .lm-topbar + .lm-body
        .lm-topbar: title, session pill, filter chips (All/Near Zone/In Zone),
                    Refresh All button (id="lmRefresh")
        .lm-body: .lm-left (card list) + .lm-right (detail panel)
    - Binds filter chip clicks → _lmActiveFilter → re-render cards
    - Binds Refresh All → _lmRefresh()
    - Calls _lmUpdateSession() and _lmRenderCards()

  _lmUpdateSession()
    - Computes current trading session from UTC hour
    - Updates #lmSession span text
    - Sessions: Asia, Pre-London, London Open, London/NY Pre-Overlap,
                London/NY Overlap Prime Time, NY Afternoon, Off-Hours

  _lmAgo(iso)
    - Converts ISO timestamp to relative label: "4s ago", "12m ago", "2h ago", "1d ago"

  _lmRenderCards()
    - Reads from global liveMonitorItems[] (populated by Phase 1 GET endpoint)
    - Applies _lmActiveFilter ("all" / "near" / "inzone")
    - Updates #lmCount and #lmSubLine
    - Renders each item as .setup-card with:
        symbol, timeframe, age, direction+arrow, setup type label
        score/confidence health bar (hi/mid/lo gradient classes)
        status badge, setup type tag
    - Binds card click → set _lmSelItem → call _lmRenderRight()
    - Auto-selects: keeps selection if still valid, else selects first item
    - Shows empty-state SVG if no items

  _lmRenderRight(item)
    - If item is null: shows "Select a setup" placeholder
    - Builds full right-panel HTML in one pass:
        Hero card: symbol, status badge, direction tags, price, proximity %
        Health circle: SVG arc with dashoffset computed from score %
        Health bars: Direction, Zone Defined, Confidence, Timeframe
        DeepSeek chat: wired via _lmBindChat(item)
        Hero actions: View Chart (opens openChartModal), Remove (calls removeLiveMonitorItem)
        MTF Module Matrix: via _lmMtfMatrix(item)
        Zone Tracker: via _lmZoneCard(item)
        Confirmation Checklist: via _lmChecklist(item)
        Event Timeline: placeholder div #lmEvents, filled by _lmLoadEvents(item.id)
    - Binds [data-chart] buttons → openChartModal(...)
    - Binds [data-rmitem] buttons → sets _lmSelItem=null, calls removeLiveMonitorItem(id)

  _lmMtfMatrix(item)
    - Reads item.selected_timeframes (default ["15m","1h","4h"]) for column headers
    - Rows: OB, FVG, FIB, BIAS
    - Derives cell values from item.setup_type (strong = matches item.timeframe, yes = present)
    - BIAS row: bull/bear/— from item.direction
    - Footer note: "Snapshot-based · Live MTF scan Phase 3"

  _lmZoneCard(item)
    - If no zone data: shows informational placeholder
    - Shows ZONE HIGH / CURRENT / ZONE LOW values using existing fmt() helper
    - Computes visual bar position: left% = (price - zone_low) / (zone_high - zone_low)
    - Clamps 2%–98%, hides price dot if price is null
    - Shows Proximity, TF, Size (zone width as %)

  _lmChecklist(item)
    - 6 checklist rows derived from item data:
        Direction Defined   — pass if item.direction set
        Zone Defined        — pass if zone_high + zone_low present
        Setup Type          — pass if item.setup_type set
        Timeframe           — pass if item.timeframe set
        Confidence Score    — pass/warn/pend from score threshold (70/40/below)
        Live Confirmation   — always pend (requires real-time MTF engine, Phase 3)

  _lmLoadEvents(itemId)
    - GET /api/live-monitor/events/<itemId> (Phase 1 endpoint)
    - Renders events in #lmEvents as .evt-list
    - Event type → dot class: "added" → added (purple), "updated" → agent (teal),
                               "removed" → alert (yellow)
    - Timestamp shown as _lmAgo() relative label
    - Graceful: shows "No events yet." or "Could not load events." on failure

  _lmBindChat(item)
    - Wires DeepSeek mock chat to the currently selected item context
    - Chips: "Enter now?", "Risks?", "Stop loss?", "Confirmation?"
    - Keyword routing: enter/buy/now → R.enter, risk/danger → R.risk,
                       stop/sl → R.stop, confirm/wait → R.confirm, else → R.dflt
    - Replies reference item.symbol, item.timeframe, item.setup_type, score, zone
    - Typing indicator: 800-1300ms delay, then shows reply
    - Chips hidden after first message

  _lmPaint()          — now just delegates to _lmRenderCards() (backwards compat)

6. WIRING SUMMARY
================================================================================

  PAGE LOAD:
    loadLiveMonitorItems() called (line 5121) → liveMonitorItems filled → _lmRenderCards()
    (no visible UI change at page load — lmCards div doesn't exist until tab is open)

  TAB SWITCH TO LIVE MONITOR:
    switchView("live") → renderLiveMonitorInto(scroll) → creates UI skeleton
    → startLiveMonitorPolling() → begins /api/watchlist/cache polling
    → loadLiveMonitorItems() → GET /api/live-monitor/items?active_only=1
    → liveMonitorItems filled → _lmRenderCards() → left cards rendered
    → auto-select first item → _lmRenderRight(item) → right panel rendered
    → _lmLoadEvents(item.id) → #lmEvents filled from DB events

  TAB SWITCH AWAY:
    switchView(other) → stopLiveMonitorPolling() → removes .lm-active from scroll

  CARD CLICK:
    setup-card click → _lmSelItem = items[idx] → _lmRenderRight(item)
    → right panel rebuilds with selected item data
    → _lmLoadEvents(item.id) called → events loaded from DB

  ADD TO LIVE MONITOR (from Scanner):
    addActiveToLiveMonitor(btn) → syncWatchlist() (legacy) + POST /api/live-monitor/items
    → on success: loadLiveMonitorItems() → liveMonitorItems updated → _lmRenderCards()
    → new card appears in left panel; if in LM view, right panel updates

  REMOVE BUTTON (hero card):
    [data-rmitem] click → _lmSelItem = null → removeLiveMonitorItem(id)
    → DELETE /api/live-monitor/items/<id> → loadLiveMonitorItems()
    → _lmRenderCards() → removed item gone; "Select a setup" shown on right

  FILTER CHIPS (All / Near Zone / In Zone):
    fchip click → _lmActiveFilter updated → _lmRenderCards() → filtered view

  REFRESH ALL:
    #lmRefresh click → _lmRefresh() → POST /api/watchlist/refresh (legacy cache)
    → _lmCache updated (order-flow data, not LM items)
