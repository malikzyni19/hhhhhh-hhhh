4. JS FUNCTIONS — NEW (Phase 2 base, commit 3a09f84)
================================================================================

  renderLiveMonitorInto(scroll)
    - Adds .lm-active to scroll (full-height layout override)
    - Renders locked UI shell: .lm-outer > .lm-topbar + .lm-body
    - Topbar: title line, session pill, TF chips, policy badges, module chips,
              filter chips, refresh interval chips, AI toggle, Refresh All button
    - Body: .lm-left (card list) + .lm-right (detail panel)
    - Binds all topbar controls (see Section 6 below)
    - Calls _lmUpdateSession() and _lmRenderCards()

  _lmUpdateSession()
    - Computes current trading session from UTC hour
    - Updates #lmSession span: Asia / Pre-London / London Open /
      London/NY Pre-Overlap / London/NY Overlap Prime Time / NY Afternoon / Off-Hours

  _lmAgo(iso)
    - ISO timestamp → relative label: "4s ago", "12m ago", "2h ago", "1d ago"

  _lmRenderCards()
    - Reads from global liveMonitorItems[] (from Phase 1 GET endpoint)
    - Applies _lmActiveFilter ("all" / "near" / "inzone") to produce filtered items[]
    - Renders each as .setup-card: symbol, timeframe, age, direction, setup type,
      score health bar (hi/mid/lo classes), status badge
    - Binds card click → _lmSelItem = items[idx]  ← FIXED in hotfix (uses filtered array)
    - Auto-selects: keeps selection if still valid, else selects first item
    - Updates #lmCount + #lmSubLine
    - Shows empty-state SVG if no items

  _lmRenderRight(item)
    - null item → "Select a setup" placeholder
    - Hero card: symbol, status badge, direction tags, price, proximity %
    - Health circle: SVG arc, circumference 144.51, dashoffset = 144.51*(1-score/100)
    - Health bars: Direction, Zone Defined, Confidence, Timeframe
    - DeepSeek chat panel: wired via _lmBindChat(item)
    - Hero actions: View Chart (→ openChartModal), Remove (→ removeLiveMonitorItem)
    - Below hero: MTF Matrix + Zone Tracker (2-column)
    - Full-width: Aggregate Market Context (6 placeholder cards)
    - Below: Confirmation Checklist + Event Timeline (2-column)
    - Binds [data-chart] → openChartModal(...)
    - Binds [data-rmitem] → _lmSelItem=null, removeLiveMonitorItem(id)

  _lmMtfMatrix(item)
    - Column headers: item.selected_timeframes || ["15m","30m","1h","4h","1d"]
    - Rows: OB, FVG, FIB, BIAS
    - Cell values derived from item.setup_type + item.timeframe + item.direction
    - Footer: "Snapshot-based · Live MTF scan Phase 3"

  _lmZoneCard(item)
    - No zone data → informational placeholder
    - ZONE HIGH / CURRENT / ZONE LOW using fmt() helper
    - Bar position: left% = clamp((price - zone_low)/(zone_high - zone_low)*100, 2, 98)
    - Stats: Proximity, TF, Size (zone width as %)

  _lmChecklist(item)
    - 6 rows: Direction Defined, Zone Defined, Setup Type, Timeframe,
              Confidence Score (pass ≥70 / warn ≥40 / pend <40),
              Live Confirmation (always pend — requires Phase 3 MTF engine)

  _lmLoadEvents(itemId)
    - GET /api/live-monitor/events/<itemId>
    - Renders events as .evt-list with type-coloured dots (added/agent/alert)
    - Timestamp via _lmAgo()
    - Graceful empty/error states

  _lmBindChat(item)
    - Mock DeepSeek chat wired to selected item context
    - Suggestion chips: "Enter now?", "Risks?", "Stop loss?", "Confirmation?"
    - Keyword routing → canned replies referencing symbol, TF, setup_type, score, zone
    - 800–1300ms typing indicator delay
    - Chips hidden after first send

  _lmPaint()
    - Now a thin delegate to _lmRenderCards() (backwards compatibility)

9. WIRING SUMMARY
================================================================================

  PAGE LOAD:
    loadLiveMonitorItems() → liveMonitorItems filled → _lmRenderCards()
    (no visible effect until LM tab is opened — lmCards div doesn't exist yet)

  TAB SWITCH TO LIVE MONITOR:
    switchView("live") → renderLiveMonitorInto(scroll) → UI skeleton created
    → startLiveMonitorPolling() → /api/watchlist/cache polling begins
    → loadLiveMonitorItems() → GET /api/live-monitor/items?active_only=1
    → liveMonitorItems filled → _lmRenderCards() → left cards rendered
    → auto-select first item → _lmRenderRight(item) → right panel rendered
    → _lmLoadEvents(item.id) → #lmEvents filled from DB

  TAB SWITCH AWAY:
    switchView(other) → stopLiveMonitorPolling() → .lm-active removed from scroll

  CARD CLICK:
    setup-card click → _lmSelItem = items[idx]  (filtered array, not global)
    → _lmRenderRight(item) → right panel rebuilds
    → _lmLoadEvents(item.id) → events loaded from DB

  ADD TO LIVE MONITOR (from Scanner):
    addActiveToLiveMonitor(btn) → syncWatchlist() (legacy) + POST /api/live-monitor/items
    → on success: loadLiveMonitorItems() → _lmRenderCards() → new card visible

  REMOVE (hero card Remove button):
    [data-rmitem] click → _lmSelItem=null → removeLiveMonitorItem(id)
    → DELETE /api/live-monitor/items/<id> → loadLiveMonitorItems() → _lmRenderCards()

  FILTER CHIPS:
    click → _lmActiveFilter updated → _lmRenderCards() → filtered view

  REFRESH ALL:
    click → _lmRefresh() → POST /api/watchlist/refresh (legacy order-flow cache)
