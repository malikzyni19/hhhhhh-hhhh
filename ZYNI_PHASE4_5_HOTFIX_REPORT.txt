2. EXACT REPAINT FIX APPLIED
================================================================================

  File:     templates/preview.html
  Function: loadLiveMonitorSessionContext()

  BEFORE:
    if(d&&d.ok&&d.session_context){
      _lmGlobalSessionContext = d.session_context;
    }

  AFTER:
    if(d&&d.ok&&d.session_context){
      _lmGlobalSessionContext = d.session_context;
      if(typeof _lmUpdateSession === "function"){
        _lmUpdateSession();
      }
      if(typeof _lmRenderRight === "function" && _lmSelItem){
        _lmRenderRight(_lmSelItem);
      }
    }

  Reason:
    loadLiveMonitorItems() and loadLiveMonitorSessionContext() both run
    asynchronously on LM open. If the right panel renders before the
    session-context fetch resolves, the Session Context tile shows fallback
    UTC text and never repaints.

    The fix calls _lmRenderRight(_lmSelItem) after the global session context
    is set, so the tile always repaints with real backend session data once
    the fetch completes — even if it completes after the initial render.

    _lmUpdateSession() is guarded with typeof check (safe if not defined).
    _lmRenderRight() is guarded with both typeof check and _lmSelItem truthy
    check (no repaint if no item is selected — avoids empty panel flash).
