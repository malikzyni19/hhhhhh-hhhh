2. TASK 1 — 7 BACKEND HEALTH HELPERS
================================================================================

  File: main.py
  Insertion point: after _lm_extract_mtf_summary, before _live_monitor_item_to_dict

  Functions added:

  _lm_latest_mtf_scan_from_snapshot(snapshot: dict) -> dict
    Safely extracts snapshot["latest_mtf_scan"]; returns {} if absent or invalid.

  _lm_get_tf_module(scan: dict, tf: str, module: str) -> dict
    Safe accessor into scan["tfs"][tf]["modules"][module].
    Returns {} on error keys or missing data.

  _lm_direction_from_item(item) -> str
    Normalizes item.direction → "bullish" | "bearish" | "neutral".
    Accepts: bull/bullish/long/up, bear/bearish/short/down.

  _lm_zone_health(item, current_price) -> dict
    Computes zone status for bullish/bearish/neutral direction.
    Returns: zone_status, zone_high, zone_low, distance_pct, inside_zone,
             near_zone, breach_risk, reason.
    zone_status values:
      "inside"      — price between zone_low and zone_high
      "near"        — within 2% outside zone (direction-aware)
      "watching"    — more than 2% outside zone
      "breach_risk" — price crossed through zone against setup direction
      "no_zone"     — zone_high/zone_low not set or no current price

  _lm_bias_alignment(scan: dict, direction: str) -> dict
    Reads 1h, 4h, 1d direction from MTF scan.
    bias_aligned = True if 1h matches direction (minimum requirement).
    context_aligned_count = count of 4h + 1d that also match (0–2).
    Returns: bias_1h, bias_4h, bias_1d, bias_aligned, context_aligned_count,
             reason.

  _lm_module_confluence(scan: dict, direction: str) -> dict
    Counts TFs where OB/FVG/FIB/Breaker have state "strong" or "yes".
    Bias module excluded (handled separately by bias_alignment).
    module_score = round(confirming_tfs / total_tfs_with_data * 100).
    Returns: module_score, confirmations, warnings, tf_summary.

  _lm_compute_health(item, _snap: dict = None) -> dict
    Main health computation. Accepts optional _snap override so refresh
    endpoints can pass the in-memory snap (with latest_mtf_scan merged)
    without requiring a DB commit first.

    Scoring model (max 100):
      base_pts = min(20, round(item.score / 100 * 20))
      zone_pts: inside=25, near=18, watching=8, no_zone=5, breach_risk=0
      bias_pts: aligned=25, neutral/None=10, opposite=0
      ctx_pts  = min(15, context_aligned_count * 8)    [4h + 1d alignment]
      mod_pts  = round(module_score / 100 * 15)
      health_score = clamp(sum, 0, 100)

    Grade:
      A = 80+, B = 65+, C = 50+, D = 35+, F = <35

    Status logic (priority order):
      "breach_risk" — zone.breach_risk is True
      "confirmed"   — score >= 75 AND bias_aligned AND zone in (inside, near)
      "inzone"      — zone_status == "inside"
      "near"        — zone_status == "near"
      "warning"     — bias_aligned == False OR raw_score < 35
      "watching"    — default

    Returns: phase, computed_at, health_score, grade, status, status_label,
             direction, zone, bias, confluence, checklist (6 rows), reasons,
             warnings, scoring breakdown.
