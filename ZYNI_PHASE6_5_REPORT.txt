3. NEW BACKEND HELPERS
================================================================================

  _lm_ai_agents_config() -> list
    METHOD A (if AI_AGENTS_JSON env set):
      Parses JSON list of agent configs from AI_AGENTS_JSON env var.
      Each entry: {id, label, provider, model, enabled, api_key_env, api_base_env}
      Reads actual keys from referenced env vars; returns has_key (bool) only.

    METHOD B fallback (individual env vars):
      OPENROUTER_API_KEY / OPENROUTER_MODEL / OPENROUTER_API_BASE
        -> id="default_openrouter", label="OpenRouter"
      OPENAI_API_KEY / OPENAI_MODEL
        -> id="openai_direct", label="ChatGPT"
      ANTHROPIC_API_KEY / ANTHROPIC_MODEL
        -> id="anthropic", label="Claude"
      GEMINI_API_KEY / GEMINI_MODEL
        -> id="gemini", label="Gemini"
      DEEPSEEK_API_KEY / DEEPSEEK_MODEL / DEEPSEEK_API_BASE
        -> id="deepseek", label="DeepSeek"
      CUSTOM_AI_API_KEY / CUSTOM_AI_MODEL / CUSTOM_AI_API_BASE
        -> id="custom", label="Custom AI"
      Always appends local_fallback (always configured=True).

    Returns: [{id, label, provider, model, configured, has_key, enabled, api_base}, ...]
    API keys NEVER returned.

  _lm_get_ai_agent_config(agent_id=None) -> dict
    Resolves agent by id.
    If agent_id is None or not found: returns first configured non-fallback agent.
    If no configured agents: returns local_fallback.
    NEVER returns actual API key.

4. PROVIDER-SPECIFIC HELPERS
================================================================================

  _lm_call_openai_compatible_agent(agent, context, user_message=None) -> dict
    Handles: openrouter, openai, deepseek, custom_openai
    Key env map:
      openrouter   -> OPENROUTER_API_KEY
      openai       -> OPENAI_API_KEY
      deepseek     -> DEEPSEEK_API_KEY
      custom_openai -> CUSTOM_AI_API_KEY
    POST to agent.api_base with OpenAI chat format.
    Headers: Authorization Bearer, Content-Type, optional HTTP-Referer / X-Title.
    timeout=20s. Returns parsed analysis dict.

  _lm_call_anthropic_agent(agent, context, user_message=None) -> dict
    Reads ANTHROPIC_API_KEY from env.
    POST to https://api.anthropic.com/v1/messages
    Headers: x-api-key, anthropic-version: 2023-06-01
    Parses content[0].text from response.
    timeout=20s.

  _lm_call_gemini_agent(agent, context, user_message=None) -> dict
    Reads GEMINI_API_KEY from env.
    POST to https://generativelanguage.googleapis.com/v1beta/models/<model>:generateContent
    Key passed as ?key= query param (not in header).
    Parses candidates[0].content.parts[0].text.
    timeout=20s.

  _lm_call_ai_provider(context, user_message=None, agent_id=None) -> dict
    Main router. Resolves agent via _lm_get_ai_agent_config().
    Dispatches:
      openrouter / openai / deepseek / custom_openai -> _lm_call_openai_compatible_agent
      anthropic -> _lm_call_anthropic_agent
      gemini    -> _lm_call_gemini_agent
      local_fallback or unconfigured -> _lm_local_ai_fallback directly
    All dispatch branches wrapped in try/except; falls back to local_fallback on error.
    Returns: {agent_id, agent_label, provider, model, configured, ok, analysis, error?}

  _lm_call_ai_agent(context, user_message=None) -> dict  [BACKWARD COMPAT]
    Single line: return _lm_call_ai_provider(context, user_message, agent_id=None)
    All Phase 6 code continues working unchanged.
    Dead code from original Phase 6 implementation removed.

5. CONSENSUS ENGINE
================================================================================

  _lm_consensus_from_agent_results(results: list) -> dict

  Input: list of dicts from _lm_call_ai_provider (one per configured agent).

  Algorithm:
    1. Separate ok results (ok=True, has analysis) from failed ones.
    2. If no ok results: use local fallback, return ok=False.
    3. Collect verdict votes across ok results.
    4. Weighted tie-break: confirmed_watch=5, watch=4, wait=3, high_risk=2, avoid=1.
    5. winner = verdict with most votes (tie-break by weight).
    6. avg_confidence = mean of ok results' confidence scores.
    7. agreement = winner_count / ok_count (0.0–1.0).
    8. Merged analysis = analysis from first result matching winner verdict.
    9. Merged confirmations/risks/invalidations/next_actions = deduplicated union
       from all ok results (capped at 6/6/6/4).
    10. agent_note summarises: "Consensus from N/M agents (...): K/N agree → verdict"

  Returns:
    {ok, verdict, confidence, summary, agent_count, ok_count, agreement,
     agents (per-agent summary), analysis (merged)}

    @login_required, no item ownership needed.
    Returns safe list from _lm_ai_agents_config():
      {ok, agents: [{id, label, provider, model, configured, has_key, enabled}], count}

  POST /api/live-monitor/items/<id>/ai-analyze          [UPDATED — Task 4]
    Body: {message?, force?, agent_id?, mode?}
    agent_id: specific agent to use (null/omitted = auto-select best)
    mode: "single" (default) | "all" (ignored for this endpoint — use ai-consensus)
    Calls _lm_call_ai_provider(context, message, agent_id=agent_id)
    latest_ai_analysis now includes: agent_id, agent_label fields.
    LiveMonitorEvent details now includes: agent_id field.

  POST /api/live-monitor/items/<id>/ai-consensus        [NEW — Task 6]
    @login_required, ownership check, active check.
    Body: {message?, force?}
    15s cooldown (more expensive than single-agent). force=true bypasses.
    Calls ALL configured+enabled non-fallback agents concurrently (sequential loop).
    If no configured agent succeeds: includes local_fallback result.
    Calls _lm_consensus_from_agent_results() on all results.
    Stores: snap["latest_ai_consensus"], snap["last_ai_consensus_at"]
    Creates LiveMonitorEvent(event_type="ai_consensus") with:
      verdict, confidence, agent_count, ok_count, agreement.
    Returns: {ok, item, consensus: {phase, computed_at, agent_count, ok_count,
              agreement, verdict, confidence, summary, agents[], analysis,
              source_context_at}}

  POST /api/live-monitor/items/<id>/ai-chat             [UPDATED — Task 7]
    Body: {message, agent_id?}
    agent_id: specific agent for this chat turn.
    Calls _lm_call_ai_provider(context, message, agent_id=agent_id).
    Returns now include: agent_id, agent_label.

  Total Live Monitor endpoints: 15
    (was 13 in Phase 6; +2: ai-providers, ai-consensus)

11. RISKS / NOTES
================================================================================

  Sequential agent calls in consensus:
    ai-consensus calls each configured agent one-by-one in a loop.
    If 3 agents × 20s timeout = up to 60s worst case.
    In practice, most calls complete in 2–5s.
    The 15s cooldown prevents accidental re-triggering.
    A future optimization could use threading for parallel calls.

  Agent selector depends on _lmAiAgents:
    The dropdown is built at render time from _lmAiAgents.
    If loadLiveMonitorAiProviders hasn't returned yet, dropdown shows only
    "Auto" and "All Agents" built-in options.
    Once providers load, re-render (triggered by the fetch callback) rebuilds it.

  No chat memory:
    Chat history is not persisted (same as Phase 6).
    Each chat call sends fresh context. Future Phase 7 could add memory.

  Provider errors are fully silent:
    Each provider call is try/excepted. A failing provider falls back gracefully.
    In consensus mode, failed providers are still listed in the agents[] array
    with ok=false so the frontend can display which agents responded.

  Backward compatibility:
    All Phase 6 code that called _lm_call_ai_agent() continues working.
    The wrapper delegates to _lm_call_ai_provider(agent_id=None) which picks
    the best configured agent automatically.
