2. OVERVIEW — TWO-LAYER AI ARCHITECTURE
================================================================================

  Layer 1: Built-in AI Brain (backend constant, never user-configurable)
    - OB/FIB/Order Flow core reasoning
    - Safety rules (analysis only, no trade execution)
    - Injected into every AI context call via _lm_builtin_ai_brain_rules()
    - NOT stored in snapshot_json (always fresh from code)
    - Cannot be overridden or removed by user instructions

  Layer 2: Custom AI Instructions (user preferences, stored in snapshot_json)
    - Extra filters on top of built-in brain
    - Stored in snapshot_json["custom_ai_instructions"] as list of dicts
    - Max 20 active instructions per item; max 300 chars each
    - Each instruction: {id, text, scope, source, created_at, is_active}
    - Soft-delete only (is_active set to False, never hard-deleted)
    - Safety-validated before saving — blocked phrase list enforced

5. TASK 2 — _lm_instruction_is_safe()
================================================================================

  Signature: (text: str) -> tuple[bool, str]
  Returns: (True, "") if safe, (False, reason_string) if blocked.

  Logic:
    t = text.strip().lower()
    For each phrase in _LM_BLOCKED_INSTRUCTION_PHRASES:
      If phrase in t: return False, "Instruction blocked: contains forbidden phrase '...'"
    Return True, ""

  Protected against:
    - Trade execution commands (place order, execute trade, open position)
    - Risk bypass commands (ignore SL, bypass risk guard, override risk)
    - Strategy modification (change strategy code, modify strategy)
    - Exchange account connection (use exchange account, use api key)
    - Dangerous behaviors (revenge trade, martingale)

  Returns: only active instructions (is_active != False) from snapshot["custom_ai_instructions"].

  Handles:

    - Filters out inactive (soft-deleted) instructions

7. TASK 4 — _lm_add_custom_ai_instruction()
================================================================================

  Signature: (row, text: str, source: str = "chat") -> tuple[dict, dict]
  Returns: (snap, instruction_dict) — no DB commit.

  Limits:
    - text stripped and capped at 300 chars
    - Max 20 active instructions per item (returns blocked if exceeded)
    - Safety check via _lm_instruction_is_safe() before adding

  Instruction dict format:
    {
      "id":         "ins_" + uuid4().hex[:8],
      "text":       "<cleaned text>",
      "scope":      "item",
      "source":     "<source>",
      "created_at": "<ISO timestamp>",
      "is_active":  True,
    }

  Returns on failure:
    (snap, {"blocked": True, "reason": "..."})

  No DB commit. Sets is_active=False (soft delete only).

9. TASK 6 — _lm_extract_instruction_from_chat()
================================================================================

  Signature: (message: str) -> dict
  Returns: {"text": str, "confidence": int (0-100)}

  Confidence scoring:
    Strong triggers (set confidence directly if no prior trigger):
      "save this instruction" -> 90
      "from now on"          -> 85
      "remember this"        -> 85
      "from now"             -> 75
      "do not mark"          -> 75
      "only consider"        -> 75
      "require confirmation" -> 75
      "be stricter"          -> 70
      "consider stronger"    -> 70
      "consider weaker"      -> 70

    Additive triggers (adds 20 to confidence):
      "alert me", "watch for", "ignore", "only alert", "filter", "add instruction"

    Penalties:
      -30 if message ends with "?" and confidence < 80
      -20 if len(stripped) < 15 and confidence < 90

    Threshold for auto-save: confidence >= 70

11. TASK 8 — THREE NEW REST ENDPOINTS
================================================================================

  GET /api/live-monitor/items/<item_id>/ai-instructions
    Auth: @login_required
    Returns: {"ok": True, "instructions": [...active only...]}

  POST /api/live-monitor/items/<item_id>/ai-instructions
    Auth: @login_required
    Body: {"text": "...", "source": "manual"}
    Validates safety, adds instruction, commits, creates "ai_instruction_added" event.
    Returns: {"ok": True, "instruction": {...}, "item": {...}}
    On blocked: {"ok": False, "blocked": True, "reason": "..."}
    On limit: {"ok": False, "blocked": True, "reason": "Max 20 active instructions..."}

  DELETE /api/live-monitor/items/<item_id>/ai-instructions/<instruction_id>
    Auth: @login_required
    Marks is_active=False, commits, creates "ai_instruction_removed" event.
    Returns: {"ok": True, "removed": True, "item": {...}}
    On not found: {"ok": False, "removed": False}

  All endpoints: 404 if item not found or not owned by current user.

12. TASK 8 — ai-chat ENDPOINT UPDATED
================================================================================

  /api/live-monitor/items/<item_id>/ai-chat — Phase 6.6 addition:

  Before sending to AI provider:
    1. _lm_extract_instruction_from_chat(user_message) -> {text, confidence}
    2. If confidence >= 70:
       a. _lm_instruction_is_safe(text) -> (safe, block_reason)
       b. If not safe:
          Return {"ok": False, "reply": "Instruction blocked: ...",
                  "instruction_saved": False, "blocked": True}
       c. If safe and within limits:
          _lm_add_custom_ai_instruction(row, text, source="chat")
          Commit. Create "ai_instruction_added" LiveMonitorEvent.
          Return {"ok": True, "reply": "Saved as a custom AI instruction: ...",
                  "instruction_saved": True, "instruction": {...}, "item": {...}}
    3. If confidence < 70: normal AI chat flow (instruction_saved: False in response)

  All ai-chat responses now include "instruction_saved": bool field.
