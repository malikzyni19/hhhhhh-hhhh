6. AUTO DETECT BEHAVIOR (Task 7)
================================================================================

  Button:    Auto (id="lmAutoDetectBtn") — in Event Timeline card header.
  State var: _lmAutoDetect (bool) — initialized from localStorage.
  Key:       localStorage["zyni_lm_auto_detect"] = "1" / "0"
  Default:   OFF

  When ON:
    - Calls _lmStartAutoDetect(item.id)
    - Runs detect-events every 60 seconds for the currently selected item only
    - Skips tick if previous request is still in-flight (_lmAutoDetectRunning flag)
    - Self-stops if _lmSelItem changes or is cleared
    - Updates #lmDetectedBlock (if events view)
    - Updates #lmDetectAge (Fresh/Stale label)
    - Calls _lmLoadEvents to refresh timeline (if events view)

  When OFF:
    - Calls _lmStopAutoDetect() — clears interval, resets flags

  Safety constraints:
    - Does NOT call ai-analyze
    - Does NOT call ai-consensus
    - Does NOT run for all items — selected item only
    - Does NOT overwrite #lmEvents when AI Instructions view is active
    - Only one interval timer at a time (clearInterval before new setInterval)

  Timer lifecycle:
    - _lmRenderRight(null) → _lmStopAutoDetect() always
    - _lmRenderRight(item) → if auto-detect ON and item ID changed → _lmStartAutoDetect(item.id)
    - _lmRenderRight(item) → if same item ID → timer not interrupted
    - Toggle OFF → _lmStopAutoDetect()
