PHASE 11.1B — MTF ORDERFLOW HISTORY ENGINE
────────────────────────────────────────────────────────────────────────────────

STATUS: COMPLETE

WHAT WAS BUILT
  Fixes Phase 11.1 limitations:
  - Multi-candle history (up to 20 candles per TF) instead of single snapshot
  - True per-candle CVD from Binance klines taker volumes instead of proxy
  - Correct parent→child TF mapping (4h uses 1h/30m/15m/5m, not just 5m)
  - Linear regression slope for series-level trend state analysis
  - SMC zone fusion: 7 zone flow states from cross-TF evidence

HELPERS IMPLEMENTED (7 functions)
  _lm_child_orderflow_timeframes   — parent TF → child TF list
  _lm_fetch_klines_with_delta      — Binance klines with k[7]/k[10] taker vol
  _lm_build_series_orderflow_state — multi-candle series via linear regression
  _lm_build_tf_orderflow_history   — single-TF history (delta+CVD series+context)
  _lm_build_mtf_orderflow_history  — orchestrates all child TFs
  _lm_build_mtf_history_summary    — compact per-TF summary for candidates
  _lm_build_smc_orderflow_fusion   — SMC zone flow state from cross-TF evidence

PARENT→CHILD TF MAPPING
  1d  → [4h, 1h, 30m, 15m]
  4h  → [1h, 30m, 15m, 5m]
  1h  → [30m, 15m, 5m]
  30m → [15m, 5m]
  15m → [5m]
  5m  → [5m]

CVD METHOD PRIORITY (best → worst)
  candle_delta                — Binance taker_buy_quote per kline (k[10])
  aligned_orderflow_snapshots — LiveMonitorCandleOrderflow.delta_net_sum (DB)
  delta_proxy                 — current 5m snapshot fallback
  unavailable                 — no data

ZONE FLOW STATES (smc_orderflow_fusion)
  defended   — 3+ bullish signals, 0 danger → behavior: touch_limit
  absorbing  — 2+ bullish signals           → behavior: confirmation
  attacked   — 2+ danger signals            → behavior: wait
  breaking   — 3+ danger signals            → behavior: block
  ignored    — 0 bullish, 0 danger          → behavior: wait
  uncertain  — insufficient/conflicting     → behavior: confirmation/wait

CANDIDATE ENRICHMENT
  Each candidate now gets:
    mtf_orderflow_history_summary  — compact per-TF state summary
    smc_orderflow_fusion           — zone flow state + evidence + behavior

EXECUTION INTELLIGENCE OUTPUT ADDITIONS
  candidates[n].mtf_orderflow_history_summary
  candidates[n].smc_orderflow_fusion
  mtf_orderflow_history  (top-level in EI dict)

ROUTES ADDED
  GET /api/live-monitor/items/<id>/mtf-orderflow-history?refresh=1

SNAPSHOT KEYS WRITTEN
  snapshot_json["latest_mtf_orderflow_history"]
  snapshot_json["latest_execution_intelligence"]["mtf_orderflow_history"]

UI CHANGES (preview.html)
  MTF Orderflow History table appended to EI card on Load.
  Columns: TF | CVD STATE | DELTA | OI | LIQ BIAS | METHOD
  Fusion state badge (zone_flow_state + behavior + confidence) shown above table.
  overflow-x scrollable for mobile. Hidden for Bias Shift items.
